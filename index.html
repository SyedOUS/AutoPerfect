Usage Guide

How to Use AutoPerfect

Launch the Smart App Panel once, then click the buttons in order. The whole workflow takes minutes.

The golden rule: You only visit File → Scripts once per session to open the panel. After that, everything runs from the panel buttons.
Step 1 — Prepare your document
Your .ai file needs four artboards named exactly SM, MD, L, and XL (uppercase). Place your master design on the MD artboard. Each artboard should have a matching layer with the same name. Save the file before running any script.
Step 2 — Launch the Smart App Panel

File → Scripts → AutoPerfect 3.0 → brix_smart_app

The dark floating panel opens above Illustrator. This is the only File menu visit you need for the entire session.

File › Scripts › AutoPerfect 3.0 › brix_smart_app
Step 3 — Click the buttons in order

Click FIX — Magic Cleanup

A dialog asks you to choose a tracking value (0, -10, -25) or skip. Make your choice. The script cleans all four artboards in place — trims spaces, rounds fonts, snaps strokes, pins content to artboard top, and crops artboard height. A summary alert appears when done.

brixMagicCleanup.jsx

Click QR — Quick Renditions

A dialog asks for tracking again (applies to MD text before scaling). Make your choice. The script normalizes MD text, cleans MD, then generates proportionally scaled SM / L / XL renditions from MD. Font rounding and stroke snapping are applied on all artboards.

brixSmartRenditions.jsx

Click QC — QC Unlocked

A brand selection dialog appears — choose your brand profile (Mounjaro, Zepbound, Kisunla, etc.). The script scans all artboards for issues and saves a styled HTML report next to your .ai file, then opens it in your browser. Review any flagged issues before exporting.

brix QC Unlocked.jsx

Click PNG's — Export PNGs

Confirm the dialog. The script removes empty layers and artboards, then exports all remaining artboards as 300 DPI PNG24 transparent files to the same folder as your .ai file. The document is saved automatically.

Script 01

Magic Cleanup

Cleans all four artboards in place — fonts, tracking, strokes, spacing, and artboard cropping. Designs must already be placed on all artboards before running this.

brixMagicCleanup.jsxv2.1All 4 ArtboardsIn-place
What makes this different from Smart Renditions: Magic Cleanup works on artwork that is already placed on all four artboards. It cleans each one in place — it does not generate or duplicate anything.
How to run

Open Smart App Panel → click FIX

A dialog appears asking you to choose a tracking value to apply to all text: 0, -10, -25, or Skip to leave existing tracking untouched. Choosing Cancel exits without running.

What it does
01

Tracking choice dialog

Choose 0, -10, -25, or Skip. This value is applied to all text frames across all four artboards during the cleanup.

02

Remove empty text frames (global)

Scans all text frames in the document and removes any that are blank or contain only whitespace. Skips locked and hidden layers.

03

Text attribute normalization (per artboard)

For each artboard layer (SM, MD, L, XL): applies your chosen tracking, resets horizontal/vertical scale to 100%, resets baseline shift to 0, resets character rotation to 0, sets kerning to Auto.

04

Font size rounding (per artboard)

Rounds every character's font size to the nearest whole number and enforces per-artboard minimums: SM 8pt, MD 10pt, L 12pt, XL 12pt.

05

Double-space cleanup

Removes consecutive double-spaces and trims leading/trailing spaces character-by-character, preserving mixed font style runs.

06

Stroke width snapping

Snaps all path stroke widths to clean values: 0.25, 0.5, 0.75, or the nearest whole number.

07

Pin content to artboard top & crop height

Shifts all content on each layer flush to the artboard's top edge, then trims the artboard height to exactly fit the content — no excess canvas below.

08

Remove empty layers

Recursively checks all layers and removes completely empty ones. Their names are listed in the final summary alert.

Script 02

Smart Renditions

Normalizes the MD master artboard, then generates proportionally scaled SM, L, and XL renditions from it — with font rounding and stroke snapping applied throughout.

brixSmartRenditions.jsxv3.4MD → SM / L / XLAuto-scale
Run Magic Cleanup first. Smart Renditions reads MD as the source. If MD has messy fonts or strokes, those will carry into all renditions. Always run FIX before QR.
How to run

Open Smart App Panel → click QR

A tracking dialog appears — choose the value to apply to MD text before scaling (0, -10, -25, or Skip). The script then runs fully automatically and shows a completion alert.

What it does
01

Tracking choice dialog

Choose your tracking value for MD text. This is applied before any scaling, so all renditions inherit the correct tracking.

02

Remove empty text frames (global)

Clears any blank text frames from the whole document before processing.

03

Normalize MD text

Applies your chosen tracking to all MD text, then resets scale (100%), baseline shift (0), character rotation (0), and kerning (Auto) on every character in the MD layer.

04

Clean MD — trim spaces & snap strokes

Removes double-spaces, trims leading/trailing spaces, and snaps all stroke widths on the MD layer.

05

Crop MD artboard to content

Shifts MD content flush to the artboard top and trims the artboard height to fit the content exactly.

06

Generate SM, L, XL renditions

Duplicates all MD content into each target layer. Scales by the ratio of each artboard's width to MD's width. After scaling, strokes are re-snapped and content is ungrouped, centred horizontally, and aligned to the artboard top.

07

Font rounding — all four artboards

Rounds every character's font size to a whole number on SM, MD, L, and XL. Enforces minimums: SM 8pt, MD 10pt, L 12pt, XL 12pt.

Font minimums per artboard
ArtboardMinimum font size

Key capabilities

Proportional scaling

Uses each artboard's width-to-MD ratio. No distortion, no manual resizing needed.

Auto artboard crop

Each rendition's artboard is trimmed to the exact content height after scaling.

Stroke re-snapping

After scaling, all strokes are re-snapped to clean values so thin lines don't become sub-pixel.

Error reporting

Non-critical errors are collected and listed in the completion alert — partial runs are always visible.

Script 04

Export PNG

Cleans up empty layers and artboards, then exports every remaining artboard as a transparent 300 DPI PNG24 — named automatically from the document and artboard names.

exportPng.jsxv11300 DPI PNG24Auto-named
Document must be saved first. The script exports to the same folder as your .ai file. If the document is unsaved, it will alert and stop.
How to run

Open Smart App Panel → click PNG's

A confirmation dialog appears: "Deleting empty Layers and Artboards. Confirm?" — click OK. The script exports all artboards, saves the document, then shows a completion alert.

What it does
01

Confirm dialog

Asks before making any changes. Clicking Cancel exits cleanly with nothing modified.

02

Delete empty layers

Recursively removes all layers and sub-layers with no content. Skips locked or hidden layers.

03

Delete empty artboards

Detects artboards with no overlapping page items and removes them before exporting.

04

Export each artboard as PNG24

Sets the active artboard, builds the filename, and exports with artboard clipping, transparency, and anti-aliasing at 300 DPI (417% scale from 72 DPI base).

05

Auto file naming

Single artboard: DocumentName.png. Multiple artboards: DocumentName ArtboardName.png. Default artboard names containing "Artboard" are stripped from the filename.

06

Save & report

Saves the .ai file after all exports complete. Shows a success alert or lists any artboards that failed with error details.

Best Practices

Tips & Best Practices

Get the most out of AutoPerfect with these workflow recommendations, dos and don'ts, and swatch/stroke references.

Dos and don'ts

Do

  • Name artboards exactly SM, MD, L, XL in uppercase
  • Design all artwork on the MD artboard only
  • Give each artboard a matching layer with the same name
  • Run FIX before QR — clean first, then generate
  • Run QC before PNG's — catch issues before exporting
  • Save the document before clicking PNG's

Don't

  • Use lowercase or mixed-case artboard names ("sm" or "Sm" will fail)
  • Place content manually on SM, L, or XL layers — QR will overwrite them
  • Run QR twice without resetting SM/L/XL — it stacks on top of previous renditions
  • Move individual script files out of the AutoPerfect 3.0 folder
  • Click PNG's on an unsaved document

Pro tips

  • FIX is safe to run multiple times — it won't double-process clean documents
  • Use "Skip" tracking in FIX if your design already has deliberate tracking set
  • QC is read-only — run it at any point during your workflow, not just at the end
  • The QC HTML report saves next to your .ai file — you can re-open it any time
  • If the panel closes, just re-launch brix_smart_app from File → Scripts

Support

Troubleshooting

Solutions to the most common issues when installing or running AutoPerfect.

Installation issues

Scripts don't appear under File → Scripts

Illustrator wasn't fully restarted after copying the folder. Fully quit (Cmd+Q on Mac, Alt+F4 on Windows) and reopen. Also check the AutoPerfect 3.0 folder is directly inside Scripts, not nested inside another subfolder.

macOS blocks the script with a security warning

Go to System Settings → Privacy & Security → click "Allow Anyway" next to the blocked file. Then run the script again and click Open.

Windows asks for administrator permission

Normal for Program Files. Click Continue. If you don't have admin rights, ask your IT team or run Illustrator as Administrator.

Runtime issues

QR stops without doing anything

All four artboards must be named exactly SM, MD, L, XL in uppercase. Check the Artboards panel and rename any that are wrong.

PNG's says "Please save the document first"

Use File → Save (Cmd/Ctrl+S) then click PNG's again.

QC report doesn't open in browser

The HTML file is still saved next to your .ai file. Look for DocumentName_QC_Report.html and double-click to open it manually.

FIX is slow on a large document

FIX operates character-by-character across every text frame. Large files may take 30–60 seconds. Do not click away — wait for the completion alert.

Panel closed unexpectedly

Simply re-launch brix_smart_app from File → Scripts → AutoPerfect 3.0. The panel will reopen at the bottom-left of your screen.

Quick checklist
  • Is Illustrator updated to the latest version?
  • Was the document saved before clicking PNG's?
  • Are artboards named SM, MD, L, XL in uppercase?
  • Was Illustrator fully restarted after installation?
  • Is the AutoPerfect 3.0 folder inside the Scripts directory (not somewhere else)?
  • Did you run FIX before QR?
