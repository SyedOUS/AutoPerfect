⚡ Script 01

Brix Smart Renditions

Automates the creation of multi-size renditions (SM, MD, L, XL) from a single master MD artboard — scaled, cleaned, and cropped in one run.

brixSmartRenditions.jsx Non-destructive Auto-scale 4 Artboards
In one sentence: Duplicates your MD master design to SM, L, and XL artboards at the correct proportional scale — with font rounding, stroke snapping, and tight artboard cropping applied automatically.
How it works
01

Setup & Validation

Checks that all four artboards named SM, MD, L, XL exist. Any missing layers are created automatically before anything else runs.

02

Empty Text Frame Cleanup

Scans every text frame across the entire document and removes any that contain only whitespace, non-breaking spaces, or are completely blank.

03

MD Master Artboard Cleanup

Removes double-spaces and trims leading/trailing spaces character-by-character. Rounds all font sizes to whole numbers enforcing a 10pt minimum. Snaps all stroke widths to clean values.

04

Move & Crop MD

Shifts all content flush to the top of the MD artboard, then trims the artboard height to exactly fit the content — no whitespace below.

05

Generate SM / L / XL Renditions

Duplicates all MD content into each target layer and scales it proportionally based on the ratio of each artboard's width to MD's width. Fonts are re-rounded with per-size minimums (SM: 8pt, L/XL: 12pt). Strokes are re-snapped.

06

Align & Crop Each Rendition

Centers content horizontally, aligns to the top edge, and crops each artboard height to fit the scaled content tightly — ready for export.

Key capabilities
🎯

Proportional Scaling

Uses each artboard's width-to-MD-width ratio to scale content correctly without distortion.

🔢

Font Rounding

Rounds every character's font size to a whole number and enforces per-artboard minimums.

✏️

Stroke Snapping

Cleans sub-pixel stroke widths to predictable values: 0.25, 0.5, 0.75, or whole integers.

✂️

Auto Artboard Crop

Trims artboard height on all four boards to the exact content boundary — ready for export.

✨ Script 02

Magic Fix

A comprehensive in-place cleanup script that fixes every artboard in the document — fonts, strokes, spacing, content alignment, and empty layer removal — in a single run.

MagicFix.jsx All Artboards In-place Cleanup No Duplication
Key difference from Smart Renditions: Magic Fix cleans and fixes all existing artboards in place. It does not duplicate or scale content — it is a cleanup-only tool designed to be run on any document at any stage.
Step-by-step process
01

Enable Scale Corners

Turns on Illustrator's Scale Corners preference so rounded corners behave correctly on any resized items.

02

Global Empty Text Frame Removal

Scans all text frames across the document. Deletes blank or whitespace-only ones. Respects locked and hidden layers.

03

Text Spacing Cleanup (Per Artboard)

Removes consecutive double-spaces and trims leading/trailing spaces character-by-character, preserving all mixed font style runs intact.

04

Font Size Enforcement (Per Artboard)

Rounds font sizes with a custom rule: fractions below 0.6 round down, 0.6 and above round up. Enforces minimums per artboard name. Only writes when a change is needed.

05

Stroke Width Snapping (Per Artboard)

Snaps all stroke widths to 0.25, 0.5, 0.75, or the nearest whole number.

06

Second Empty Frame Pass

Re-checks for text frames that became empty after the space cleanup and removes them.

07

Move Content to Top & Crop

Shifts all content flush to the artboard top, then resizes the artboard bottom to fit the content height exactly.

08

Remove Empty Layers

Recursively checks every layer and removes completely empty ones. Reports their names in a final alert.

🔍 Script 03

Brix QC Report

Scans the entire Illustrator document and generates a detailed, styled HTML quality-check report — identifying font, spacing, stroke, swatch, and bounds issues per artboard.

brix_QC_Unlocked.jsx HTML Report Saves to Disk Read-only scan
Output: Saves a DocumentName_QC_Report.html next to the .ai file and opens it in the browser automatically. Makes no changes to the Illustrator document.

How it works
01

QC User Cache

On first run, prompts for the reviewer's name and saves it to userData/Illustrator_QC_User.txt. Subsequent runs load it automatically.

02

Per-Artboard Scanning

For each artboard, collects all text frames and path items. Checks every character's font size and spacing, and every stroke's width against the valid values list.

03

Bounds & Overflow Detection

Uses a normalized overlap-area algorithm to assign each item to its artboard, then flags any item whose visible bounds extend beyond the artboard by more than 0.0001pt.

04

Swatch Audit

Checks every swatch against known bad-name patterns: "New Swatch", CMYK strings like "C=0 M=0…", RGB strings, hex values, and unnamed defaults.

05

HTML Report Generation

Builds a complete styled HTML report with a summary dashboard, per-artboard sections with badge counts, and highlights the exact word containing a bad font size in context.

📊

🔤 Script 04

Text Normalization

Resets all text attributes across every text frame in the document — tracking, kerning, scale, baseline shift, and character rotation — via a quick choice dialog.

text_cleanup.jsx Quick Dialog Style Preservation Character-level
How to use: Run the script — a dialog appears asking you to choose a tracking value. Click your option and the script processes all text frames and shows a summary alert.
Tracking options
ButtonTracking ValueUse Case
Set Tracking 00Standard reset to default
Set Tracking -10-10Slightly tighter for medium sizes
Set Tracking -25-25Tight tracking for large display text
What gets reset
↔️

Tracking

Sets letter-spacing to your chosen value (0, -10, or -25) on every character individually.

📏

Scale Reset

Both horizontal and vertical character scale are restored to exactly 100%.

⬆️

Baseline Shift

Any baseline offset is zeroed out across all characters.

🔄

Character Rotation

Individual character rotation is reset to 0° — removes any per-character tilt applied manually.

🔡

Kerning → Auto

Forces kerning method to AutoKernType.AUTO, removing all manual or optical overrides.

Important notes
  • Processes every text frame on every layer — no per-artboard or per-layer filter.
  • Font size, font family, color, and weight are not touched — only the 5 attributes listed above.
  • Mixed-style runs are preserved because the script operates character-by-character.
  • Performs two redraws before and after to ensure Illustrator commits all changes visually.
  • Executes an undo step on load as a hard state reset to avoid Illustrator state conflicts.
📤 Script 05

Export PNG

Cleans up empty layers and artboards, then exports every remaining artboard as a transparent 300 DPI PNG24 file — named, saved, and done in one click.

exportPng.jsx 300 DPI PNG24 Transparent Auto-named
Requirement: The document must be saved before running — the script uses the document's save location as the export destination.
Export process
01

Confirm Dialog

Asks "Deleting empty Layers and Artboards. Confirm?" before making any changes. Cancelling exits cleanly.

02

Delete Empty Layers

Recursively removes all layers and sub-layers with no page items. Skips locked or hidden layers.

03

Delete Empty Artboards

Checks each artboard for visible content overlap. Any artboard with no overlapping items is removed.

04

Export Each Artboard as PNG

Sets the active artboard, constructs the filename, and exports using PNG24 with artboard clipping, transparency, and anti-aliasing.

05

File Naming

Single artboard: DocumentName.png. Multiple artboards: DocumentName ArtboardName.png. Default artboard names containing "Artboard" are omitted.

06

Save & Report

Saves the Illustrator document after export. Shows a completion alert or lists any artboards that failed.

GitHub Repository Setup

How to host this documentation on GitHub Pages and ensure the download button works correctly for every visitor.

Repository file structure
Place the ZIP file in the same folder as this HTML file. The download button uses a relative path — AutoPerfect_2_0.zip — so both files must live in the same directory.

Step-by-step GitHub Pages setup

📋 Complete Setup Instructions

  1. Create a new GitHub repository — go to github.com → New Repository. Name it anything, e.g. autoperfect-docs. Set it to Public.
  2. Rename this HTML file to index.html — GitHub Pages serves index.html as the homepage automatically.
  3. Upload both files to the repository root — click Add file → Upload files and drag in:
    • 📄 index.html
    • 📦 AutoPerfect_2_0.zip
  4. Enable GitHub Pages — go to your repo → Settings → Pages → under Source, select Deploy from a branch → choose main branch → folder / (root) → click Save.
  5. Wait ~1 minute then visit https://yourusername.github.io/autoperfect-docs/ — your site is live.
  6. Test the download button — click "Download All Scripts" in the sidebar. The browser should download AutoPerfect_2_0.zip directly.
✅ Why this works: The download button uses href="AutoPerfect_2_0.zip" download="AutoPerfect_2_0.zip" — a relative path. Since both files sit in the same root directory, GitHub Pages resolves the ZIP correctly and the download attribute forces the browser to download rather than open it.
What NOT to do
  • Do not put the ZIP in a subfolder like /assets/ unless you update the href in the download button to match.
  • Do not use a private repository — GitHub Pages requires a public repo on the free plan.
  • Do not rename the ZIP file without also updating the href and download attributes in the sidebar footer button.
  • Do not use GitHub's web editor to rename index.html — it's safer to rename it locally first then upload.
